# VibeOps: Agentic Development Onboarding System

**Transform developers from traditional to agentic development mindset through verification-first practices**

## 🎯 What This Is

A comprehensive onboarding system that teaches **VibeOps** - the philosophy of human-AI collaborative software development. This system embodies verification-first development principles when code is abundant and judgment is scarce.

## 🏛️ The Four Pillars of VibeOps

1. **AI Partnership Over Tool Usage** - Collaborate with AI agents as thinking partners
2. **Verification Over Implementation** - Prove before you build
3. **Orchestration Over Coding** - Guide intelligent systems, don't just write code
4. **Emergence Over Planning** - Create conditions for solutions to arise

## 🚀 Quick Start

1. **Read the Philosophy**:
   ```bash
   cat AGENTIC_DEVELOPMENT_MANIFESTO.md
   ```

2. **Start the Guided Tutorial**:
   ```bash
   ./onboard.sh
   ```

3. **Add VibeOps to Existing Project**:
   ```bash
   ./setup-current-project.sh /path/to/your/project
   ```

## 📋 System Components

### Core Files
- **`AGENTIC_DEVELOPMENT_MANIFESTO.md`** - Complete VibeOps philosophy
- **`onboard.sh`** - Master orchestrator for the learning journey
- **`setup-current-project.sh`** - Integrates VibeOps into existing projects

### Tutorial System
- **`tutorial-phases/`** - 5-phase learning journey:
  - Phase 0: Philosophy Foundation
  - Phase 1: Frame the Problem
  - Phase 2: Model Security & Threats
  - Phase 3: Generate with AI Partnership
  - Phase 4: Verify & Observe

### Verification System
- **`verification-gates/`** - Production-ready verification pipelines:
  - `security-pipeline.sh` - OWASP LLM Top 10, secrets detection
  - `quality-pipeline.sh` - Coverage, complexity, maintainability
  - `deployment-pipeline.sh` - Canary, health checks, rollback

### Integration Tools
- **`manifesto-integration/`** - Tools to embed VibeOps in any project
- **`shared/`** - Reusable components and utilities
- **`examples/`** - Reference implementations

## 🎓 Learning Journey

### Complete Tutorial Flow
```
Philosophy Foundation → Assessment Gate →
Frame Problems → Acceptance Tests Gate →
Model Threats → Security Gate →
Generate with AI → Code Verification Gate →
Verify & Observe → Production Readiness Gate →
🎉 Graduation: Agentic Developer
```

### Key Learning Outcomes
- **Verification-first thinking** - Tests and specs before implementation
- **Human-AI partnership** - Collaborative problem-solving patterns
- **Security-first development** - Threat modeling integrated from day one
- **Observable systems** - Built-in monitoring and rollback capabilities

## 📊 Verification Gates

Every phase includes mandatory verification gates:
- **80% philosophy assessment** before advancing
- **All acceptance tests passing** before code generation
- **Complete threat model approved** before implementation
- **Full security pipeline passing** before deployment
- **Observable production system** before graduation

**Core Principle**: No progression without verified understanding.

## 🔄 Error Handling & Recovery

**Graceful Degradation**:
- No Claude Code? → Conceptual walkthrough mode
- Agent generation fails? → Manual coding with verification gates
- Security tools missing? → Checklist-based security review
- Deployment issues? → Local development with mock observability

**Safety Nets**:
- Sandboxed tutorial environment
- Git checkpoints at every gate passage
- "Nuclear reset" option for any phase
- Backup verification methods when tools fail
